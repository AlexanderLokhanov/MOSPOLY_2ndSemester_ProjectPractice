# Ход выполнения проектной практики

В период прохождения проектной практики была проведена работа, направленная на освоение современных инструментов разработки программного обеспечения, а также получение практических навыков создания и сопровождения программных проектов.

Практика выполнялась в соответствии с поставленным заданием и включала работу с системой контроля версий Git, оформление документации и разработку собственного программного решения.

## Основные этапы работы

### 1. Изучение задания и подготовка рабочей среды

На первом этапе были изучены требования к проекту, структура репозитория и порядок оформления отчётной документации.  
Также была выполнена подготовка рабочей среды: настройка необходимых инструментов разработки и подключение репозитория проекта.

### 2. Работа с Git и GitHub

В ходе практики были закреплены навыки работы с системой контроля версий Git.  
Были освоены:
- создание и ведение репозитория;
- фиксация изменений с помощью commit;
- загрузка проекта на GitHub.

Использование Git позволило удобно отслеживать этапы разработки и поддерживать актуальное состояние проекта.

### 3. Подготовка документации проекта

Отдельное внимание уделялось ведению документации.  
Все отчётные материалы оформлялись в формате Markdown, что позволило структурировать информацию и обеспечить удобство чтения документации.

### 4. Разработка статического веб-сайта

В рамках практики был создан статический веб-сайт, посвящённый проекту по дисциплине «Проектная деятельность».

На сайте были реализованы следующие разделы:
- главная страница с кратким описанием проекта;
- раздел «О проекте»;
- страница с информацией об участниках;
- журнал хода выполнения работы;
- страница с полезными ресурсами и ссылками.

При создании сайта были использованы базовые технологии веб-разработки: HTML и CSS.

### 5. Разработка Telegram-бота для учёта посещаемости

Основной частью практической работы стала разработка Telegram-бота, предназначенного для автоматизации учёта посещаемости студентов.

Функционал бота позволяет преподавателю отправлять запрос о присутствии на занятии, после чего бот автоматически собирает ответы пользователей и формирует информацию о посещаемости.

В процессе разработки были реализованы:
- обработка команд преподавателя;
- автоматическая рассылка сообщений;
- получение и обработка ответов студентов;
- фиксация статуса присутствия;
- логика взаимодействия пользователей с ботом.

Разработка данного проекта позволила получить практический опыт создания прикладных Telegram-ботов и работы с пользовательскими запросами.

### 6. Оформление итогового отчёта

На заключительном этапе были собраны и систематизированы результаты выполненной работы.  
Итоговый отчёт был оформлен в соответствии с требованиями практики и подготовлен для сдачи.
