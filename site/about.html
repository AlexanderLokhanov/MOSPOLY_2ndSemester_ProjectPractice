О проекте

AttendBot

Telegram-бот для учёта посещаемости: администратор запускает опрос, студенты отвечают одной кнопкой, причины отсутствия сохраняются автоматически

Проблема

Ручной учёт — это боль

  • Преподаватели тратят 5–10 минут на перекличку каждой пары
  • Бумажные журналы теряются, портятся и не дают быстрой аналитики
  • Причины отсутствия нигде не фиксируются систематически
  • Нет единой базы данных для анализа посещаемости по группе
  • Ручной ввод данных в деканат занимает дополнительное время
Решение

Опрос через Telegram за секунды

  • Администратор запускает опрос одной командой
  • Бот рассылает вопрос всем зарегистрированным студентам
  • Студент отвечает нажатием одной кнопки — «Да» или «Нет»
  • При ответе «Нет» бот запрашивает причину и сохраняет её
  • Все данные мгновенно попадают в базу данных
Сценарий использования

Как проходит опрос

Шаг 1

Администратор запускает опрос

Пользователь с правами администратора (проверяется по ADMIN_ID из конфига) отправляет команду /poll. Бот проверяет права и начинает рассылку.

Шаг 2

Рассылка вопроса

Бот отправляет всем пользователям, хоть раз написавшим боту, вопрос «Вы на паре?» с двумя инлайн-кнопками: Да ✅ и Нет ❌.

Шаг 3

Ответ и причина

При нажатии «Да» — отметка сохраняется сразу. При «Нет» — бот просит написать причину отсутствия текстом. Ответ фиксируется в таблице attendance.

Архитектура

Как устроен бот

Модуль 1

Обработчики Telegram

Модуль main.py содержит все хендлеры бота: регистрацию новых пользователей при первом обращении (/start), проверку прав администратора, запуск рассылки командой /poll и обработку кнопок «Да» / «Нет».

Модуль 2

Сбор причины отсутствия

Когда студент нажимает «Нет», бот переводит диалог в состояние ожидания текста (FSM). Следующее сообщение пользователя воспринимается как причина и сохраняется в поле reason таблицы attendance.

Модуль 3

Рассыльщик

Модуль broadcaster.py получает список всех chat_id из таблицы users и последовательно отправляет каждому сообщение с инлайн-клавиатурой через aiogram.

Модуль 4

База данных

Модуль database.py управляет двумя таблицами SQLite: users (все когда-либо написавшие боту) и attendance (отметки посещаемости с причиной отсутствия и временем ответа).
