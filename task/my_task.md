# Проект: AttendBot — система учёта посещаемости

---

# Введение

## Цель проекта

Разработать Telegram-бота для автоматизации учёта посещаемости студентов на занятиях и упрощения работы преподавателя.

## Задачи

- изучить Telegram Bot API и принципы работы Telegram-ботов;
- реализовать систему регистрации пользователей;
- добавить механизм массовой рассылки опросов;
- реализовать сбор ответов студентов;
- организовать хранение данных в базе SQLite;
- создать статический сайт проекта;
- подготовить документацию и визуализацию архитектуры.

# Исследование предметной области

## Что такое система учёта посещаемости

Система учёта посещаемости - это программное решение, позволяющее автоматически фиксировать присутствие пользователей и сохранять данные для дальнейшего анализа.

## Основные компоненты системы

| Компонент | Описание |
| --- | --- |
| Telegram Bot | Основной интерфейс взаимодействия |
| Пользователь | Студент, отвечающий на опрос |
| Администратор | Преподаватель, запускающий опрос |
| SQLite | База данных для хранения информации |
| FSM | Механизм ожидания причины отсутствия |
| Broadcast System | Система массовой рассылки |

## Схема работы системы

```text
Преподаватель → /poll → AttendBot
                         ↓
            Рассылка всем студентам
                         ↓
                 Да ✅ / Нет ❌
                         ↓
             Сохранение результата
```

## Telegram Bot API

Telegram Bot API предоставляет инструменты для создания ботов, которые могут:

- отправлять сообщения;
- получать ответы пользователей;
- работать с кнопками;
- обрабатывать команды;
- хранить и анализировать данные.

### Иллюстрация 1 (работа опроса)

```text
/poll
   ↓
Бот рассылает сообщение:
"Вы сейчас на паре?"

Да ✅ → присутствует
Нет ❌ → ввод причины
```

# Техническое руководство

### Шаг 1. Создание проекта

Была создана структура проекта AttendBot:

```text
src/
 ├── main.py
 ├── broadcaster.py
 ├── database.py
 ├── config.py
 └── attendbot.db
```

### Шаг 2. Создание Telegram-бота

Основной модуль бота:

```python
bot = Bot(token=BOT_TOKEN)
dp = Dispatcher(storage=MemoryStorage())
```

Бот использует библиотеку `aiogram` для асинхронной обработки сообщений.

### Шаг 3. Регистрация пользователей

При первом запуске пользователь автоматически добавляется в базу данных:

```python
@dp.message(Command("start"))
async def cmd_start(message: Message):
    db.add_user(
        chat_id=message.chat.id,
        username=message.from_user.username or "",
        first_name=message.from_user.first_name or "",
    )
```

#### Иллюстрация 2 (регистрация пользователя)

```text
Пользователь → /start
               ↓
      users(chat_id, username)
```

### Шаг 4. Реализация опроса

Преподаватель запускает опрос командой:

```python
@dp.message(Command("poll"))
async def cmd_poll(message: Message):
```

После этого бот отправляет сообщение всем зарегистрированным пользователям.

### Шаг 5. Работа с кнопками

Для ответа используются inline-кнопки:

```python
InlineKeyboardButton(
    text="Да ✅",
    callback_data="present_yes"
)
```

Если студент нажимает «Нет», бот переводит пользователя в состояние ожидания причины отсутствия.

#### Иллюстрация 3 (FSM)

```text
Нажатие "Нет"
      ↓
WaitingReason
      ↓
Ввод причины
      ↓
Сохранение в БД
```

### Шаг 6. База данных SQLite

Для хранения информации используются две таблицы:

```sql
users
attendance
```

Таблица `users` хранит зарегистрированных пользователей, а таблица `attendance` — данные о посещаемости.

#### Иллюстрация 4 (структура БД)

```text
users
 ├── chat_id
 ├── username
 └── first_name

attendance
 ├── user_id
 ├── present
 ├── reason
 └── marked_at
```
